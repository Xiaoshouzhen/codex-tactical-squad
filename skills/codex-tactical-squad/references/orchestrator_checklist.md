# Orchestrator Checklist

1. Restate the task and decide whether the triad should be used.
2. Create or update the tactical position registry for the session.
3. Create a task intake.
4. Reuse an existing planner if suitable; otherwise launch one.
5. Ask the planner for a round contract and wait for the planner submission.
6. Tighten ambiguous scope or acceptance criteria before execution.
7. Reuse an existing executor for the current task cluster if suitable; otherwise launch one.
8. Send the approved contract to the executor with explicit ownership and wait for the executor submission.
9. Launch a fresh reviewer unless there is a specific reason to reuse one.
10. Send the task, contract, and executor report to the reviewer and wait for the reviewer submission.
11. Send the review result to the planner and wait for the planner decision submission.
12. Accept only if there are no material findings or residual risk is explicitly accepted.
13. If needed, run a narrow follow-up round instead of reopening the whole task.
